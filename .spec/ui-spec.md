# `kaname` UI仕様書 (ui-spec.md)

## 1. フロントエンド構成方針
本システムで生成されるナレッジベースは、Quartz v5 を用いて静的公開される。ユーザーインターフェースはObsidianライクなナレッジグラフベースのデザインを採用するが、ビルド負荷および描画パフォーマンスを担保するため、特定の機能に対して制限を設ける。

---

## 2. 画面レイアウトおよび基本機能

### 2.1 ナビゲーションおよびサイドバー
- **左サイドバー**
  - **Explorer:** Vault内のフォルダ・ファイル階層（SSoT組織別、コンセプト別、レポート別）をツリー表示する。
  - **Search:** Quartzネイティブの高速全文検索ウィンドウ（ショートカット `Ctrl + K`）を配置する。
- **右サイドバー**
  - **Table of Contents (TOC):** アクティブなMarkdownファイルの見出し階層を動的に抽出し、画面スクロールに追従する形で目次を配置する。
  - **Backlinks:** 表示中のトピックにリンクしている他のOFMファイル群を自動抽出し、「参照元（被リンク）」として一覧表示する。

### 2.2 メインコンテンツエリア
- **Obsidian Flavored Markdown (OFM) の忠実なレンダリング**
  - `[[Internal Link]]` は自動的にQuartz内部の相対リンクへと変換され、ホバー時にプレビューカードを表示する。
  - プロパティ（YAML Frontmatter）で記述されたタグ（`#tag`）やステータス情報をヘッダー部に洗練されたバッジとして描画する。

---

## 3. UIの厳格な制限事項

### 3.1 Graph Viewの完全無効化
- **デザインルール**
  - インタラクティブなノート間のネットワークを可視化する Graph View (ローカルグラフ・グローバルグラフ問わず) は、ビルド速度向上およびCPU負荷軽減のため、設定レベルで完全にオフ（無効化）とする。
  - UI上にグラフ表示のアイコンや描画領域は一切レンダリングしない。
